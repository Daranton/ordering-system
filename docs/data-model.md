# Data Model

## OrderStatus

An enum of the valid states an order can be in. Defined in `src/domain/order.py`.

| Value       | Meaning                          |
|-------------|----------------------------------|
| `pending`   | Order received, not yet acted on |
| `confirmed` | Order accepted                   |
| `shipped`   | Order dispatched                 |
| `delivered` | Order received by customer       |
| `cancelled` | Order cancelled                  |
| `disputed`  | Order under dispute              |

`OrderStatus` extends `str` and `Enum`, so a status value can be used directly as a string (e.g. in JSON serialisation).

### Terminal statuses

`TERMINAL_STATUSES` in `src/domain/state_machine.py` defines which statuses are final:

```python
TERMINAL_STATUSES = frozenset({OrderStatus.CANCELLED, OrderStatus.DELIVERED})
```

An order in a terminal status cannot be updated — `Order.transition_to()` raises `InvalidTransitionError` if attempted.

| Field          | Type    | Notes                    |

| `product_name` | `str`   |                          |
| `quantity`     | `int`   |                          |

## Order

The core domain object. Defined as a Python dataclass in `src/domain/order.py`.

| Field           | Type              | Notes                                      |
|-----------------|-------------------|--------------------------------------------|
| `id`            | `str`             | UUID generated by the application service  |
| `customer_name` | `str`             |                                            |
| `status`        | `OrderStatus`     | Starts as `PENDING`                        |
| `total`         | `float`           | Sum of `quantity × unit_price` per item    |
| `created_at`    | `datetime`        | UTC timestamp set at creation              |
| `items`         | `list[OrderItem]` | At least one item required                 |

### Behaviour

- **`transition_to(new_status)`** — updates `status`. Raises `InvalidTransitionError` if the current status is in `TERMINAL_STATUSES`.

## Persistence

Orders are stored in PostgreSQL via SQLAlchemy. Two tables are used:

### `orders`

| Column        | Type           | Notes                        |
|---------------|----------------|------------------------------|
| `id`          | `VARCHAR(36)`  | Primary key (UUID)           |
| `customer_name` | `VARCHAR(100)` |                            |
| `status`      | `ENUM`         | PostgreSQL enum type         |
| `total`       | `NUMERIC(10,2)`|                              |
| `created_at`  | `DATETIME`     |                              |
| `deleted_at`  | `DATETIME`     | `NULL` unless soft-deleted   |

Indexed on `status` and `created_at`.

### `order_items`

| Column         | Type           | Notes                              |
|----------------|----------------|------------------------------------|
| `id`           | `INTEGER`      | Primary key, auto-increment        |
| `order_id`     | `VARCHAR(36)`  | Foreign key → `orders.id`          |
| `product_name` | `VARCHAR(200)` |                                    |
| `quantity`     | `INTEGER`      |                                    |
| `unit_price`   | `NUMERIC(10,2)`|                                    |

ORM models are defined in `src/infrastructure/db/models.py`. The repository in `src/infrastructure/db/repositories/order_repository.py` translates between these models and the domain `Order` / `OrderItem` dataclasses.

## API Schemas

Pydantic models in `src/api/schemas.py` that represent orders at the HTTP boundary.

| Schema            | Used for                               |
|-------------------|----------------------------------------|
| `OrderCreate`     | POST `/orders` request body            |
| `OrderUpdate`     | PATCH `/orders/{id}` request body      |
| `OrderResponse`   | Response body for all order endpoints  |
| `OrderItemSchema` | Nested line item in Create and Response|

`OrderUpdate.status` accepts case-insensitive strings (`"PENDING"`, `"Shipped"`) and normalises them to lowercase before enum validation.
