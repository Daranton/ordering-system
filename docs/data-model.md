# Data Model

## OrderStatus

An enum of the valid states an order can be in. Defined in `src/utils/models.py`.

| Value       | Meaning                        |
|-------------|--------------------------------|
| `pending`   | Order received, not yet acted on |
| `confirmed` | Order accepted                 |
| `shipped`   | Order dispatched               |
| `delivered` | Order received by customer     |
| `cancelled` | Order cancelled                |
| `disputed`  | Order under dispute            |

`OrderStatus` extends `str` and `Enum`, so a status value can be used directly as a string (e.g. in JSON serialisation).

## Order

The core domain object. Defined as a Python dataclass in `src/utils/models.py`.

| Field      | Type          | Default          | Notes                          |
|------------|---------------|------------------|--------------------------------|
| `order_id` | `str`         | required         | UUID generated by `ids.py`     |
| `customer` | `str`         | required         |                                |
| `item`     | `str`         | required         |                                |
| `quantity` | `int`         | required         | Must not be negative           |
| `status`   | `OrderStatus` | `PENDING`        | Coerced from string if needed  |
| `amount`   | `float`       | `0.0`            |                                |

Validation in `__post_init__`:
- `quantity < 0` raises `ValueError`
- A string `status` is coerced to `OrderStatus` automatically

## Persistence

Orders are stored as a JSON file at `data/orders.json`. The format is a flat dict keyed by order ID:

```json
{
  "a1b2c3d4-...": {
    "customer": "Alice",
    "item": "Widget",
    "quantity": 2,
    "status": "pending",
    "amount": 0.0
  }
}
```

`load_orders` skips corrupt entries with a warning rather than crashing. `save_orders` creates parent directories if they don't exist.

## API Schemas

Pydantic models in `src/api/schemas.py` that represent orders at the API boundary. Separate from the `Order` dataclass - they validate HTTP input and shape HTTP output.

| Schema          | Used for                        |
|-----------------|---------------------------------|
| `OrderCreate`   | POST request body (new order)   |
| `OrderUpdate`   | PATCH request body (update)     |
| `OrderResponse` | Response body (outgoing JSON)   |
| `OrderItemSchema` | Nested line item (reused in Create and Response) |

`OrderUpdate.status` accepts case-insensitive strings (`"PENDING"`, `"Shipped"`) and normalises them to lowercase before enum validation.
